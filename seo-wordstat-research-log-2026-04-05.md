# SEO: Wordstat / Keyword Planner Research Log

Дата: 2026-04-05

## 1. Зачем нужен этот файл

Этот файл нужен для второго слоя research после ручного SERP-review.

Если SERP-review отвечает на вопрос:

- `какой тип страницы ранжируется`
- `насколько сильны конкуренты`

то Wordstat / Keyword Planner отвечает на вопрос:

- `есть ли у запроса достаточный спрос`
- `какие формулировки реально используют люди`
- `какие long-tail варианты стоит брать в первую волну`

Этот файл нужен, чтобы не потерять связь между:

- частотностью
- конкурентностью
- интентом
- реальным SEO-решением

## 2. Как использовать

Рабочий порядок:

1. Открыть [seo-keyword-research-sheet-2026-04-05.csv](/Users/ibragimibragimov/Documents/Мои%20проекты/Portfolio_project/seo-keyword-research-sheet-2026-04-05.csv)
2. Взять сначала приоритетные RU-кластеры:
   - `Audit / Conversion`
   - `B2B`
   - `SaaS`
   - `Pricing`
3. Для каждого приоритетного запроса проверить:
   - Wordstat
   - Google Keyword Planner, если доступен
4. Сюда записывать не только цифры, но и вывод

## 3. Что фиксировать по каждому запросу

По каждому запросу желательно зафиксировать:

- основной запрос
- сервис, где проверялся спрос
- точная или приблизительная частотность
- сезонность, если заметна
- похожие формулировки
- long-tail варианты
- уточнения по интенту
- итоговое решение

## 4. Как принимать решение по запросу

### `GO`

Ставить, если одновременно верно следующее:

- интент совпадает с нужным типом страницы
- есть заметный спрос или сильный long-tail потенциал
- выдача не выглядит полностью закрытой сильными брендами

### `GO LATER`

Ставить, если:

- интент подходит
- спрос есть
- но конкуренция тяжёлая
- или сначала нужен supporting cluster

### `HOLD`

Ставить, если:

- спрос сомнительный
- интент размытый
- запрос уводит не в тот тип услуги
- страница под него рискует каннибализироваться

## 8. Как читать итог после заполнения

После заполнения этого файла должны стать ясны 4 вещи:

1. Есть ли у `Audit / Conversion` не только хороший SERP, но и достаточный спрос.
2. Действительно ли `B2B` стоит держать вторым кластером.
3. Есть ли у `SaaS` жизнеспособный long-tail в RU, или его нужно отложить полностью.
4. Есть ли смысл рано выпускать pricing-статью.
