Плюсы:

- есть отдельный текстовый слой для LLM/AI-систем

Важно:

- `llms.txt` полезен как дополнительный слой, но не заменяет реальные entity-сигналы, страницы кейсов и внешние подтверждения

## 6. Что сейчас остаётся проблемой

### 6.1. Не закрыта entity-связность

Самая заметная проблема:

- `sameAs` в JSON-LD всё ещё пустой
- в footer соцссылки пока стоят как `href="#"`

Это означает:

- у сайта нет нормальной привязки к реальным публичным профилям
- для поисковиков и AI-систем это слабое место в подтверждении сущности/личного бренда

### 6.2. В проекте есть placeholder-ссылки

Сейчас в HTML ещё встречаются `href="#"`:

- logo
- project cards
- footer social links

Почему это плохо:

- это шум для аудитов
- это неидеально для crawlability и UX
- это выглядит как недоделанные link targets

### 6.3. Single-page архитектура ограничивает рост

Даже при хорошем SEO-слое текущий сайт остаётся одним основным URL.

Это значит:

- он может быть хорош для бренда
- он может ловить часть long-tail запросов
- но ему трудно масштабироваться под несколько коммерческих кластеров сразу

### 6.4. Нет отдельных SEO-активов

Сейчас у сайта нет:

- отдельных страниц услуг
- отдельных страниц кейсов
- отдельных статей/гайдов

Это главный стратегический потолок проекта.

### 6.5. Есть незакрытые содержательные заглушки

По проекту всё ещё не закрыты:

- реальные соцссылки
- реальные скриншоты проектов
- реальное фото
- Formspree ID

Не всё из этого относится к SEO напрямую, но это влияет на trust, conversion и E-E-A-T.

## 7. Что уже можно считать хорошим уровнем

На текущий момент сайт уже можно считать:

- технически SEO-подготовленным на хорошем базовом уровне
- заметно лучше среднего портфолио по meta + JSON-LD + FAQ + robots/sitemap

Но пока нельзя считать:

- полностью законченным по entity SEO
- готовым к серьёзной конкуренции по коммерческим запросам
- полноценной SEO-системой, а не сильной брендовой landing page

## 8. Главный вывод

Технический слой в проекте уже в хорошем состоянии.

Главный следующий рост лежит не в ещё одном meta-теге, а в:

- entity-сигналах
- нормальных ссылках вместо заглушек
- отдельных индексируемых страницах
- кейсах как самостоятельных URL
- контенте под long-tail и коммерческие интенты

Текущий проект — это хороший SEO-ready фундамент.
Но для роста нужен следующий архитектурный шаг.
