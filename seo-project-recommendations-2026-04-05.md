# SEO: Конкретные предложения и план действий

Дата: 2026-04-05

## 1. Главный вывод

Для этого проекта SEO стоит строить в два этапа:

1. Довести текущий single-page сайт до чистого, завершённого состояния.
2. Превратить сайт в маленькую SEO-систему с отдельными URL под услуги, кейсы и статьи.

Без второго этапа сайт может быть сильным брендовым активом, но ему будет трудно расти по небрендовым коммерческим запросам.

### 1.1. Рабочее решение по рынку и языку

На этапе 1 для проекта принимается стратегия:

- `RU-first`

Это означает:

- SEO в ближайшие 3–6 месяцев строится под русскоязычный рынок
- текущая русскоязычная версия сайта остаётся основной SEO-базой
- EN-направление не отменяется, но выносится за пределы первого SEO-этапа

Практический смысл:

- сначала делаем research, URL-структуру, service pages, case pages и статьи под RU SERP
- международных клиентов на этом этапе логичнее привлекать не через SEO, а через другие каналы
- к EN-версии возвращаемся только после первичного результата по RU и первых данных из Search Console

## 2. Что сделать обязательно в ближайшее время

### 2.1. Закрыть entity SEO

Сделать:

- добавить реальные ссылки в `sameAs`
- заменить `href="#"` в footer social links на реальные профили

Желательные профили:

- GitHub
- LinkedIn
- X / Twitter

Если есть:

- Product Hunt
- Behance / Dribbble
- Clutch / Upwork / Contra / Wellfound

Почему это важно:

- это основной способ связать сайт с реальной сущностью в интернете
- это усиливает trust и entity consistency

### 2.2. Убрать placeholder-ссылки

Заменить или переработать:

- logo link
- project cards
- footer social links

Если страницы/кейсы ещё не готовы, лучше:

- временно сделать карточки не ссылками
- либо вести на реальные case pages, когда они появятся

### 2.3. Проверить live-версию после деплоя

После публикации обязательно:

- подключить Google Search Console
- отправить sitemap
- проверить индексируемость главной
- проверить Rich Results Test
- проверить Schema Markup Validator

## 3. Средний SEO-пакет для этого проекта

Это оптимальный объём работы без перегруза.

Состав:

- 1 брендовая главная
- 3 страницы услуг
- 3 страницы кейсов
- 3 статьи

Важно:

- порядок публикации теперь определяется не общим списком идей, а первым SERP-review
- на текущем этапе подтверждён приоритет `Audit / Conversion`
- `B2B` остаётся в плане как второй кластер
- `SaaS` уходит в отложенный слой до дополнительного long-tail research

## 4. Какие страницы услуг делать первыми

### 4.1. `uslugi/audit-i-redizain-saita-na-konversiyu/`

Роль страницы:

- первый подтверждённый коммерческий кластер
- напрямую поддерживает hero и CTA `Запросить разбор сайта`

Что продавать:

- аудит текущего сайта
- поиск точек потери доверия и заявок
- редизайн/пересборка страницы или сайта
- понятный следующий шаг после разбора

Почему первая:

- лучше всего совпадает с текущим оффером сайта
- в SERP заметен коммерческий интент
- выдача допускает вход для нового домена через сильную, взрослую service page

### 4.2. `uslugi/razrabotka-b2b-saitov-i-veb-interfeisov/`

Роль страницы:

- коммерческий кластер под B2B и продуктовые интерфейсы

Что продавать:

- B2B-сайты
- интерфейсы для сложного оффера
- digital presence для компаний с длинным циклом сделки

Почему вторая:

- кластер рабочий, но SERP тяжелее
- в выдаче много сильных агентских страниц, цен, кейсов и блоков функционала
- логичнее идти туда после первого audit-кластера, а не параллельно с ним

### 4.3. `uslugi/razrabotka-saitov-dlya-saas/`

Роль страницы:

- отложенный кластер под SaaS и startup-направление

Под что она нужна:

- сайты для SaaS
- product marketing sites
- launch pages
- conversion-focused SaaS websites

Почему не первая:

- RU SERP по head-term выглядит шумно и расплывчато
- интент пересекается с MVP, стартапами и разработкой SaaS-продуктов
- перед выпуском страницы нужен отдельный long-tail research

## 5. Какие кейсы делать первыми

### 5.1. `cases/kova-technology-ecommerce/`

Фокус:

- e-commerce
- UX
- конверсия

Почему первым:

- лучше всего поддерживает первый кластер `Audit / Conversion`
- помогает доказать работу с доверием, структурой и результатом

### 5.2. `cases/saas-dashboard-redesign/`

Фокус:

- SaaS
- продуктовый интерфейс
- скорость / performance / usability

Почему вторым:

- хороший supporting case для B2B/SaaS слоя
- не обязателен в первой волне, если главная задача сейчас — audit-кластер

### 5.3. `cases/outdoor-photography-portfolio/`

Фокус:

- брендовая визуальная подача
- кастомный frontend
- storytelling

## 6. Какие статьи делать первыми

### 6.1. `blog/pochemu-sait-ne-konvertiruet/`

Тема:

- почему сайт не конвертирует
- где теряются заявки
- что влияет на доверие и решение пользователя

Зачем:

- поддерживает аудит/разбор сайта
- подтверждён как сильная informational page по первому SERP-review

### 6.2. `blog/chto-dolzhno-byt-na-b2b-sajte/`

Тема:

- обязательные блоки B2B-сайта
- что должно быть на странице, чтобы она продавала

Зачем:

- поддерживает B2B service page

### 6.3. `blog/skolko-stoit-razrabotka-saita-dlya-saas-i-b2b/`

Тема:

- стоимость разработки
- диапазоны бюджета
- от чего зависит цена

Зачем:

- собирает коммерческий long-tail
- фильтрует неподходящих лидов

## 7. Как связать эти страницы между собой

Минимальная схема внутренней перелинковки:

- главная → все страницы услуг
- главная → все кейсы
- главная → ключевые статьи
- каждая услуга → релевантные кейсы
- каждая статья → услуга + кейс
- каждый кейс → услуга

Это нужно не только для SEO, но и для конверсии.

## 8. Приоритетный план работ

### Этап 1. Доведение текущего сайта

Сделать:

- `sameAs`
- соцссылки
- убрать placeholder links
- live-проверка индексации

### Этап 2. Архитектурное решение

Сделать:

- выбрать способ многостраничности
- не выпускать 10 страниц вручную без шаблонной структуры
- подготовить основу под service page, case page и article

Опорный документ:

- [seo-multipage-architecture-plan-2026-04-05.md](/Users/ibragimibragimov/Documents/Мои%20проекты/Portfolio_project/seo-multipage-architecture-plan-2026-04-05.md)

### Этап 3. Первый кластер: Audit / Conversion

Сделать:

- страницу `audit-i-redizain-saita-na-konversiyu`
- кейс `kova-technology-ecommerce`
- статью `pochemu-sait-ne-konvertiruet`
- связать их с главной и между собой

### Этап 4. Второй кластер: B2B

Сделать:

- страницу `razrabotka-b2b-saitov-i-veb-interfeisov`
- статью `chto-dolzhno-byt-na-b2b-sajte`
- релевантный supporting case

### Этап 5. Третий кластер: SaaS / Pricing

Сделать:

- дополнительный long-tail research по SaaS
- только после этого решать, нужен ли head-term URL под SaaS
- отдельно опубликовать pricing-материал

### Этап 6. Работа по данным

После 2–6 недель:

- смотреть Search Console
- искать страницы с показами и низким CTR
- смотреть, какие запросы появляются
- усиливать именно те URL, которые уже начали набирать показы

## 9. Что не делать

- не пытаться бесконечно улучшать только главную страницу
- не плодить много слабых страниц
- не писать общие статьи “про разработку вообще”
- не придумывать метрики и результаты, которые нельзя подтвердить
- не считать, что schema и meta сами по себе выведут сайт в топ

## 10. Самая практичная стратегия для этого проекта

Если кратко:

- текущий сайт оставить как сильную брендово-конверсионную главную
- дальше строить SEO вокруг страниц услуг, кейсов и статей

Это лучший компромисс между объёмом работы и шансом на реальный рост.

## 11. Что можно считать успехом на первом этапе

Через первый нормальный цикл работы успехом будет:

- сайт хорошо индексируется
- появляются показы не только по бренду
- service pages начинают получать impressions
- статьи получают long-tail трафик
- кейсы начинают работать как trust pages

Только после этого имеет смысл масштабировать контент дальше.
