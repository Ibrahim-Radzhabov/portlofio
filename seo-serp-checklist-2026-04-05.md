# SEO: Ручной SERP-чеклист

Дата: 2026-04-05

## 1. Зачем нужен этот файл

Этот чеклист нужен для ручной проверки выдачи по приоритетным запросам.

Его задача:

- понять, кто уже занимает топ
- понять, какой тип страниц ранжируется
- понять, реалистично ли заходить в запрос новым сайтом
- не строить контент-план вслепую

## 2. Когда использовать

Использовать после выбора рынка и языка.

Для текущего проекта:

- этап 1 = `RU-first`

Значит сначала анализируются русскоязычные запросы.

## 3. Сколько запросов проверять

На первом проходе достаточно:

- 5 запросов минимум
- 10 запросов оптимально

Лучше взять:

- 2 запроса из кластера `Audit / Conversion`
- 2 запроса из кластера `B2B`
- 2 запроса из кластера `SaaS`
- 1–2 запроса из кластера `Pricing`

## 4. Что смотреть по каждому запросу

### 4.1. Базовая информация

- сам запрос
- дата проверки
- устройство: desktop / mobile
- регион, если важен

### 4.2. Общий вид выдачи

Отметить:

- есть ли AI Overview
- есть ли блок “Люди также спрашивают”
- есть ли видео
- есть ли карты / локальный блок
- есть ли рекламные объявления
- есть ли feature snippets

### 4.3. Кто в топ-10

По каждому запросу посмотреть:

- агентства
- фрилансеры
- каталоги
- статьи
- маркетплейсы услуг
- корпоративные сайты

Главный вопрос:

`Какой тип страницы Google считает лучшим ответом на этот интент?`

### 4.4. Тип ранжирующихся страниц

Отметить, что именно ранжируется:

- homepage
- service page
- category page
- article
- case study
- directory / aggregator

Это критично.

Если ранжируются только статьи, не надо сразу пытаться лезть туда service page.
Если ранжируются только service pages, не надо надеяться, что статья заберёт коммерческий интент.

### 4.5. Сила конкурентов

Проверить визуально:

- насколько сильный бренд в топе
- есть ли крупные агентства
- есть ли сайты с очень большим портфолио
- есть ли заметно слабые или устаревшие страницы

Особенно важно:

- есть ли в топе кто-то, кого реально можно обойти качеством страницы

### 4.6. Паттерны контента

Посмотреть топ-3–5 результатов и отметить:

- как сформулирован title
- что стоит в H1
- какие H2 повторяются у нескольких конкурентов
- есть ли FAQ
- есть ли кейсы
- есть ли блок стоимости
- есть ли CTA
- есть ли отзывы

Главный вопрос:

`Что считается “нормой” для этой выдачи, без чего страницу нельзя туда выпускать?`

### 4.7. Интент

По итогу по каждому запросу зафиксировать:

- informational
- commercial
- transactional
- mixed

Если интент смешанный, это важно отдельно отметить.

### 4.8. Реалистичность для нового домена

Оценить вручную:

- высокая
- средняя
- низкая

Логика:

- высокая: в топе есть страницы, которые можно обойти качеством и фокусом
- средняя: запрос хороший, но конкуренты сильные
- низкая: топ забит очень сильными доменами, агрегаторами или брендами

## 5. Шаблон оценки одного запроса

Используйте такой формат:

### Запрос: `[вставить запрос]`

- Кластер:
- Интент:
- Тип выдачи:
- Что в топе:
- Какие типы страниц ранжируются:
- Общие паттерны titles/H1:
- Общие паттерны контента:
- Что повторяется у конкурентов:
- Есть ли слабые места у топа:
- Реалистичность для нового домена:
- Вывод:
  - идти
  - отложить
  - искать более узкий вариант

## 6. Какие выводы должны появиться после проверки

После 5–10 запросов у вас должны появиться не “ощущения”, а решения:

- какие 1–2 кластера запускать первыми
- какие запросы слишком широкие
- где лучше заходить через service page
- где лучше заходить через статью
- где нужен кейс как supporting page
- какие слова реально используют конкуренты в titles и H1

## 7. Красные флаги

Если видите это в выдаче, нужно быть осторожнее:

- весь топ занят крупными агрегаторами
- почти весь топ — крупные агентства с сильным брендом
- страницы очень сильные и насыщены кейсами, отзывами, FAQ, ценами и ссылками
- интент не совпадает с той страницей, которую вы планировали делать

В таких случаях лучше:

- искать более длинный запрос
- менять угол захода
- или понижать приоритет кластера

## 8. Зелёные флаги

Если видите это, запрос может быть хорошей точкой входа:

- в топе есть слабые страницы
- страницы не очень глубокие
- мало хороших кейсов
- нет сильной специализации под ваш сегмент
- есть смешанная выдача и можно сделать более точный ответ под интент

## 9. Практический порядок проверки

### Шаг 1

Взять 5–10 приоритетных запросов из:

- `seo-keyword-research-sheet-2026-04-05.csv`

### Шаг 2

По каждому пройтись по чеклисту выше.

### Шаг 3

После этого обновить:

- `seo-project-recommendations-2026-04-05.md`
- `seo-url-map-2026-04-05.md`

Но только на основе того, что реально видно в SERP.

## 10. Главный принцип

SERP важнее предположений.

Если выдача показывает, что запрос требует:

- статью,
- а не service page,

то план нужно менять.

Если выдача показывает, что по запросу реально ранжируются:

- нишевые страницы,

то это может быть хорошей точкой входа даже для нового домена.
