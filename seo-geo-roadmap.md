# SEO / GEO Дорожная карта
*Полное руководство по оптимизации сайта для поисковиков и ИИ-систем*

---

## Как читать этот документ

**Слой 1 — Технический** → разработчик делает один раз при запуске. Занимает 1–2 дня.
**Слой 2 — Стратегический** → вы делаете постоянно, месяцами. Именно он определяет попадёте ли вы в ответы ИИ.

**Главная ошибка:** думать, что Слой 1 достаточен. Технический слой — это разрешение на участие. Стратегический — это победа.

### Обозначения

| Иконка | Значение |
|---|---|
| 🔴 Критично | Без этого всё остальное не работает |
| 🟠 Важно | Сильно влияет на результат |
| 🟡 Желательно | Даёт дополнительное преимущество |
| ⏱ быстро | До 30 минут |
| ⏱⏱ средне | 1–3 часа |
| ⏱⏱⏱ долго | Полдня и больше |

## Перед началом: определите тип проекта

От типа зависит набор schema.org и приоритеты:

| Тип | Основная schema | Ключевые секции |
|---|---|---|
| **Портфолио фрилансера** | `Person` + `Service` | FAQ, Review, CreativeWork |
| **Агентство / студия** | `Organization` + `Service` | FAQ, Review, AggregateRating |
| **E-commerce** | `Organization` + `Product` + `Offer` | Product, Review, BreadcrumbList |
| **SaaS / продукт** | `SoftwareApplication` + `Organization` | FAQ, AggregateRating, HowTo |
| **Блог / медиа** | `Person` + `Article` | BlogPosting, BreadcrumbList, FAQPage |

## Слой 1 — Технический

### 1.1 🔴 Meta-теги ⏱

Первое что читают краулеры. Без правильных meta — всё остальное теряет смысл.

**Title — формула:**
```
[Имя/Бренд] — [Профессия/Услуга] | [Ключевой дифференциатор]
```
Пример: `Дейл Макманус — Фулстек-разработчик | 15+ лет высоконагруженных веб-продуктов`

Правила:
- Длина: 50–60 символов
- Самое важное — в начале
- Не дублировать дословно в og:title

**Description — формула:**
```
[Кто вы] + [что делаете] + [для кого] + [ключевой результат или дифференциатор]
```
Пример: `Фулстек-разработчик с 15-летним опытом. Создание кастомных сайтов и веб-приложений для бизнеса, который серьёзно относится к конверсии.`

Правила:
- Длина: 140–160 символов
- Содержать глагол действия
- Не повторять title дословно
- Не спамить ключами

**Полный набор:**
```html
<!-- Основные -->
<title>...</title>
<meta name="description" content="...">
<link rel="canonical" href="https://домен.com">
<meta name="robots" content="index, follow, max-snippet:-1, max-image-preview:large, max-video-preview:-1">
<meta name="author" content="Имя">

<!-- Open Graph -->
<meta property="og:title" content="...">
<meta property="og:description" content="...">
<meta property="og:type" content="website">
<meta property="og:image" content="https://домен.com/og-image.png">
<meta property="og:url" content="https://домен.com">
<meta property="og:locale" content="ru_RU">
<meta property="og:site_name" content="...">

<!-- Twitter Card -->
<meta name="twitter:card" content="summary_large_image">
<meta name="twitter:title" content="...">
<meta name="twitter:description" content="...">
<meta name="twitter:image" content="https://домен.com/og-image.png">

<!-- Мобильные -->
<meta name="theme-color" content="#ваш-цвет">
<meta name="msapplication-TileColor" content="#ваш-цвет">
```

**Не делать:**
- Keyword stuffing: `разработчик сайтов Москва купить заказать дёшево` — поисковики понижают
- Пустой description — браузер вырежет случайный кусок текста со страницы
- Одинаковый title на всех страницах многостраничного сайта

### 1.2 🔴 JSON-LD Structured Data ⏱⏱

Самый важный GEO-сигнал. Именно его читают ИИ-системы чтобы понять кто вы и что предлагаете.

**Архитектура: всегда используйте `@graph`**

`@graph` позволяет описать несколько связанных сущностей в одном блоке и ссылаться между ними через `@id`. Это намного эффективнее отдельных schema-блоков.

#### Для портфолио / фрилансера:

```json
{
  "@context": "https://schema.org",
  "@graph": [
    {
      "@type": "ProfilePage",
      "dateModified": "2025-01-01",
      "inLanguage": "ru",
      "mainEntity": { "@id": "https://домен.com/#person" }
    },
    {
      "@type": "Person",
      "@id": "https://домен.com/#person",
      "name": "Полное имя",
      "alternateName": "Name in Latin",
      "jobTitle": "Профессия",
      "description": "2–3 предложения: кто вы, что делаете, для кого, ключевой результат.",
      "url": "https://домен.com",
      "email": "email@домен.com",
      "image": "https://домен.com/og-image.png",
      "knowsAbout": ["навык 1", "технология 2", "специализация 3"],
      "knowsLanguage": ["ru", "en"],
      "hasOccupation": {
        "@type": "Occupation",
        "name": "Профессия",
        "skills": "React, Node.js, TypeScript, ..."
      },
      "makesOffer": [
        {
          "@type": "Offer",
          "itemOffered": {
            "@type": "Service",
            "@id": "https://домен.com/#service-1",
            "name": "Название услуги",
            "description": "Что входит, для кого, какой результат.",
            "serviceType": "Web Development",
            "provider": { "@id": "https://домен.com/#person" },
            "areaServed": { "@type": "Place", "name": "Весь мир" }
          }
        }
      ],
      "sameAs": [
        "https://linkedin.com/in/username",
        "https://github.com/username",
        "https://x.com/username"
      ]
    },
    {
      "@type": "WebSite",
      "@id": "https://домен.com/#website",
      "url": "https://домен.com",
      "name": "Название сайта",
      "inLanguage": "ru",
      "author": { "@id": "https://домен.com/#person" }
    }
  ]
}
```

#### Добавлять по мере готовности контента:

**Отзывы (Review):**
```json
{
  "@type": "Review",
  "author": {
    "@type": "Person",
    "name": "Имя клиента",
    "jobTitle": "Должность",
    "worksFor": { "@type": "Organization", "name": "Компания" }
  },
  "reviewBody": "Полный текст отзыва дословно, как на сайте.",
  "itemReviewed": { "@id": "https://домен.com/#service-1" },
  "reviewRating": {
    "@type": "Rating",
    "ratingValue": 5,
    "bestRating": 5
  }
}
```

**Портфолио (CreativeWork):**
```json
{
  "@type": "CreativeWork",
  "name": "Название проекта",
  "description": "Задача, решение, измеримый результат.",
  "creator": { "@id": "https://домен.com/#person" },
  "keywords": ["React", "Next.js", "E-commerce"],
  "genre": "Веб-приложение"
}
```

**FAQ (FAQPage) — только если есть видимая FAQ-секция на странице:**
```json
{
  "@type": "FAQPage",
  "mainEntity": [
    {
      "@type": "Question",
      "name": "Вопрос дословно, как на странице?",
      "acceptedAnswer": {
        "@type": "Answer",
        "text": "Ответ дословно, как на странице."
      }
    }
  ]
}
```

**Не делать:**
- Добавлять FAQPage schema без видимой FAQ-секции в HTML — Google игнорирует
- Ставить рейтинг 5/5 фейковым отзывам — риск санкций
- Копировать одну schema для разных страниц без изменений
- Оставлять `sameAs: []` пустым — это главная связь между профилями в интернете

### 1.3 🔴 FAQ-секция ⏱⏱

ИИ-системы предпочитают страницы где вопрос уже задан и отвечен. FAQ — это прямой ответ на то, что клиент мог бы спросить ИИ.

**Как выбрать вопросы:**
Думайте не "что мне важно рассказать", а "что клиент спросит у ChatGPT перед тем как нанять меня".

**Хорошие вопросы:**
- С чего начинается работа над проектом?
- Сколько стоит [услуга]? (конкретные цифры или диапазоны)
- Сколько времени занимает разработка?
- Какие технологии вы используете?
- Предоставляете ли поддержку после сдачи?
- Работаете ли с клиентами из других городов / стран?
- Как вы работаете: поэтапно или сразу весь проект?

**Плохие вопросы:**
- "Почему стоит выбрать именно вас?" — рекламный, не информационный
- "Вы лучший разработчик?" — нет конкретного ответа
- "Как заказать?" — слишком очевидно

**Требования к ответам:**
- 2–4 предложения — не слишком коротко, не эссе
- Конкретика: цифры, сроки, условия
- Дублировать точно те же вопросы и ответы в FAQPage schema

**HTML-реализация:** используйте `<details>/<summary>` — семантично, доступно, без лишнего JS.

### 1.5 🟠 llms.txt ⏱

Новый стандарт (2024). Файл в корне сайта — краткая сводка для языковых моделей. Аналог robots.txt, но вместо разрешений — содержание.

```
# [Имя / Название компании]

> [1–2 предложения: кто вы, что делаете, для кого, ключевой результат]

## Услуги
- **[Услуга 1]** — [краткое описание и для кого]
- **[Услуга 2]** — [краткое описание и для кого]

## Стек / Экспертиза
- [Технология, специализация, инструмент]

## Портфолио
- **[Проект]** — [задача] → [измеримый результат]. Стек: [...]
- **[Проект]** — [задача] → [измеримый результат]. Стек: [...]

## Стоимость
- [Услуга] — от [сумма]
- [Услуга] — от [сумма]

## Процесс работы
1. [Шаг 1]
2. [Шаг 2]

## Клиенты говорят
- **[Имя, Должность, Компания]** — «[Цитата]»

## Контакт
- Сайт: https://домен.com
- Email: email@домен.com
- [Другие каналы]

## Языки
- [Язык 1], [Язык 2]
```

**Правила:** пишите как для человека, который вас не знает. Конкретика важнее красоты. Измеримые результаты важнее описаний процесса.

### 1.6 🟠 sitemap.xml ⏱

```xml
<?xml version="1.0" encoding="UTF-8"?>
<urlset xmlns="http://www.sitemaps.org/schemas/sitemap/0.9"
        xmlns:xhtml="http://www.w3.org/1999/xhtml">
  <url>
    <loc>https://домен.com/</loc>
    <lastmod>2025-01-01</lastmod>
    <changefreq>monthly</changefreq>
    <priority>1.0</priority>
    <xhtml:link rel="alternate" hreflang="ru" href="https://домен.com/"/>
  </url>
  <!-- для многостраничных сайтов — добавить каждую важную страницу -->
</urlset>
```

**После публикации:** отправьте sitemap вручную:
- Google Search Console → Sitemaps → добавить URL
- Bing Webmaster Tools → Sitemaps

### 1.7 🟠 IndexNow — быстрая индексация ⏱

Новый протокол (Bing, Yandex, Seznam). Уведомляет поисковики об изменениях мгновенно, без ожидания краулера.

1. Сгенерировать ключ: [indexnow.org](https://www.indexnow.org/documentation)
2. Положить файл `{ваш-ключ}.txt` в корень сайта
3. Отправить запрос при каждом обновлении:

```
GET https://api.indexnow.org/indexnow?url=https://домен.com/&key={ваш-ключ}
```

Google пока не поддерживает, но использует Google Search Console → URL Inspection → Request Indexing.

### 1.8 🟠 OG-изображение ⏱⏱

Стандарт: **1200 × 630 px**, PNG.

**Что должно быть на изображении:**
- Имя / название бренда — крупно
- Профессия / ключевая услуга
- Визуальный стиль совпадает с сайтом (цвета, шрифты)
- Домен сайта

**Не делать:**
- Белый фон + чёрный текст — выглядит как Word-документ
- Мелкий текст — на превью 300px ничего не видно
- Только логотип без контекста — непонятно кто это
- Стоковые фотографии

**Инструменты:** Figma, Pillow (Python), Satori (JS), Vercel OG.

**Проверка:** [opengraph.xyz](https://www.opengraph.xyz) — как выглядит при шаринге в Telegram, Twitter, LinkedIn.

### 1.10 🟡 Производительность ⏱

```html
<!-- Font preload — уменьшает FCP (First Contentful Paint) -->
<link rel="preconnect" href="https://fonts.googleapis.com" crossorigin>
<link rel="preload" as="style" href="https://fonts.googleapis.com/css2?...">

<!-- Ресурсные подсказки для критических ресурсов -->
<link rel="preload" as="image" href="/og-image.png">
```

**Инструменты проверки:** PageSpeed Insights, Lighthouse в Chrome DevTools.
**Цель:** LCP < 2.5s, CLS < 0.1, FID < 100ms.

## Schema по типам бизнеса — примеры

### Агентство / студия

Используйте `Organization` вместо `Person`. Ключевые отличия: `AggregateRating` вместо отдельных `Review`, `Employee` для команды, `foundingDate`.

```json
{
  "@context": "https://schema.org",
  "@graph": [
    {
      "@type": "Organization",
      "@id": "https://домен.com/#org",
      "name": "Название студии",
      "alternateName": "Studio Name in Latin",
      "description": "...",
      "url": "https://домен.com",
      "email": "email@домен.com",
      "foundingDate": "2018",
      "numberOfEmployees": { "@type": "QuantitativeValue", "value": 8 },
      "logo": "https://домен.com/logo.png",
      "image": "https://домен.com/og-image.png",
      "knowsAbout": ["Web Design", "Branding", "UI/UX"],
      "hasOfferCatalog": {
        "@type": "OfferCatalog",
        "name": "Услуги",
        "itemListElement": [
          {
            "@type": "Offer",
            "itemOffered": {
              "@type": "Service",
              "@id": "https://домен.com/#service-design",
              "name": "Веб-дизайн",
              "description": "..."
            }
          }
        ]
      },
      "aggregateRating": {
        "@type": "AggregateRating",
        "ratingValue": 4.9,
        "reviewCount": 24,
        "bestRating": 5
      },
      "sameAs": ["https://linkedin.com/company/...", "https://behance.net/..."]
    }
  ]
}
```

**Важно:** `aggregateRating` работает только если отзывы реальные и их число соответствует действительности. Google проверяет.

### E-commerce (магазин)

Для магазина ключевые типы: `Product`, `Offer`, `BreadcrumbList`. `AggregateRating` на уровне магазина и на уровне каждого товара.

```json
{
  "@context": "https://schema.org",
  "@graph": [
    {
      "@type": "Store",
      "@id": "https://домен.com/#store",
      "name": "Название магазина",
      "url": "https://домен.com",
      "description": "...",
      "currenciesAccepted": "RUB",
      "paymentAccepted": "Cash, Credit Card",
      "priceRange": "₽₽",
      "aggregateRating": {
        "@type": "AggregateRating",
        "ratingValue": 4.8,
        "reviewCount": 156,
        "bestRating": 5
      }
    },
    {
      "@type": "Product",
      "name": "Название товара",
      "description": "...",
      "image": ["https://домен.com/product-1.jpg"],
      "brand": { "@type": "Brand", "name": "Бренд" },
      "sku": "SKU-001",
      "offers": {
        "@type": "Offer",
        "url": "https://домен.com/product/slug",
        "priceCurrency": "RUB",
        "price": "2990",
        "availability": "https://schema.org/InStock",
        "seller": { "@id": "https://домен.com/#store" },
        "shippingDetails": {
          "@type": "OfferShippingDetails",
          "shippingRate": {
            "@type": "MonetaryAmount",
            "value": "0",
            "currency": "RUB"
          },
          "deliveryTime": {
            "@type": "ShippingDeliveryTime",
            "handlingTime": { "@type": "QuantitativeValue", "minValue": 1, "maxValue": 2, "unitCode": "DAY" },
            "transitTime": { "@type": "QuantitativeValue", "minValue": 2, "maxValue": 5, "unitCode": "DAY" }
          }
        }
      },
      "aggregateRating": {
        "@type": "AggregateRating",
        "ratingValue": 4.7,
        "reviewCount": 43,
        "bestRating": 5
      }
    }
  ]
}
```

**Для e-commerce особенно важно:** обновлять `price` и `availability` в реальном времени. Устаревшие данные → санкции от Google.

### Блог / медиа (для каждой статьи)

На странице статьи — обязательно `Article` или `BlogPosting`:

```json
{
  "@type": "BlogPosting",
```

## Как писать кейсы для GEO

Кейс — самый эффективный формат внешнего контента для попадания в ответы ИИ. ИИ предпочитает конкретные истории с измеримыми результатами над общими описаниями услуг.

### Структура кейса

```
1. ЗАГОЛОВОК (формула)
   "[Измеримый результат] для [тип клиента] за [срок]"
   Пример: "Конверсия +40% для e-commerce за 6 недель"

2. КОНТЕКСТ (1 абзац)
   - Кто клиент (отрасль, размер, задача)
   - Почему это было сложно / нетривиально
   - Что было до вас

3. ЗАДАЧА (1–2 абзаца)
   - Конкретная проблема с цифрами
   - Почему существующие решения не подходили
   - Ограничения: бюджет, срок, технический долг

4. РЕШЕНИЕ (2–3 абзаца)
   - Что именно вы сделали (с техническими деталями)
   - Почему выбрали этот подход, а не другой
   - С какими трудностями столкнулись и как решили

5. РЕЗУЛЬТАТ (цифры обязательны)
   - Измеримые метрики до и после
   - Срок достижения результата
   - Неожиданные побочные эффекты (хорошие)

6. ВЫВОД
   - Что применимо для других похожих проектов
   - Чему вы научились
   - Когда этот подход не подойдёт
```

### Хороший кейс vs плохой

| Плохо | Хорошо |
|---|---|
| "Мы сделали красивый сайт для клиента" | "Переработали фронтенд за 6 недель: LCP упал с 4.2s до 1.1s, конверсия +38%" |
| "Использовали современные технологии" | "Выбрали Next.js вместо CRA: SSG дал -60% TTFB без изменения UX" |
| "Клиент доволен результатом" | "CEO написал: 'Впервые за 3 года наш сайт быстрее конкурентов'" |
| "Проект выполнен в срок" | "Уложились в 6 недель вместо плановых 8 — вынесли дизайн-ревью на более ранний этап" |

### Где публиковать

| Площадка | Аудитория | Формат |
|---|---|---|
| **VC.ru** | Предприниматели, стартапы | Бизнес-кейс с цифрами |
| **Хабр** | Разработчики, технари | Технический разбор + код |
| **Telegram-канал** | Ваша аудитория | Сжатая версия кейса, 300–500 слов |
| **LinkedIn** | Профессионалы | Краткий кейс + ссылка на полную статью |
| **Medium (EN)** | Международная аудитория | Полная версия на английском |

## Что делать если GEO не работает

### Диагностика — идите по этому списку сверху вниз

**Шаг 1: Проверьте индексацию**
```
site:ваш-домен.com
```
Если страницы не показываются — сайт не проиндексирован. Причины: robots.txt блокирует, нет sitemap, молодой домен (< 3 месяцев), дублированный контент.

**Шаг 2: Проверьте schema**
Google Rich Results Test → введите URL → должно быть 0 ошибок.
Частые ошибки: обязательные поля пропущены, `FAQPage` без видимого FAQ, несоответствие текста на странице и в schema.

**Шаг 3: Проверьте AI-краулеры**
```bash
# Проверьте логи сервера на наличие этих User-Agent:
GPTBot, OAI-SearchBot, anthropic-ai, ClaudeBot, PerplexityBot
```
Если их нет — либо блокируете в robots.txt, либо сайт ещё не обнаружен. Дайте 2–4 недели после деплоя.

**Шаг 4: Проверьте авторитетность**
Введите в Ahrefs или Semrush: сколько внешних сайтов ссылаются на вас?
- 0 бэклинков → сайт "невидим" для алгоритмов
- Решение: начните со Слоя 2 (платформы, контент, отзывы)

**Шаг 5: Проверьте контент**
Спросите у ChatGPT или Perplexity то, по чему хотите появляться.
- Кто появляется вместо вас? Чем их контент отличается?
- Они на крупных платформах (Clutch, Upwork)? Присутствуйте там же.
- У них больше внешних упоминаний? Публикуйте больше кейсов.

### Типичные сроки

| Действие | Когда ждать результат |
|---|---|
| Технический слой | 2–6 недель (индексация) |
| Брендовые запросы (ваше имя) | 1–3 месяца |
| Нишевые запросы (ваша специализация) | 3–9 месяцев |
| Конкурентные запросы (общие) | 6–18 месяцев |
| Попасть в ответы ИИ по нише | 6–12 месяцев при активном Слое 2 |

**Главный вывод:** если через 3 месяца после деплоя при запросе вашего имени в ChatGPT ничего нет — проблема не в технике, а в отсутствии внешних упоминаний.

## Слой 2 — Стратегический (вы делаете)

### 2.1 🔴 Профили на платформах — сделать сразу после деплоя

ИИ связывает личность/бренд по совпадению данных на разных платформах. Если везде разные описания — ИИ не может построить связный профиль.

**Правило единообразия:** везде одинаковые:
- Имя (точно как на сайте, в том числе на латинице)
- URL сайта
- Описание (можно варьировать, но суть одна)
- Фото

| Платформа | Приоритет | Что заполнить |
|---|---|---|
| **LinkedIn** | 🔴 Критично | Полный профиль + ссылка на сайт в About + все проекты в Experience |
| **GitHub** | 🔴 Критично | Активный профиль + README с описанием + реальные репозитории |
| **Clutch** | 🟠 Важно | Профиль агентства/фрилансера + первые 2–3 отзыва |
| **Google Business** | 🟠 Важно | Для локального бизнеса — карточка компании |
| **Behance / Dribbble** | 🟡 Желательно | Для дизайн-проектов |
| **VC.ru / Хабр** | 🟡 Желательно | Русскоязычная аудитория и авторитетность |
| **Product Hunt** | 🟡 Желательно | Для SaaS и продуктов |

**После заполнения:** добавить все ссылки в `sameAs` в JSON-LD.

---

### 2.2 🔴 Внешний контент — постоянно

Это самое важное для попадания в не-брендовые запросы. ИИ цитирует то, о чём написано в интернете — не только то, что написано на вашем сайте.

**Форматы которые работают лучше всего:**

**Кейсы (самый высокий ROI):**
```
Структура:
1. Проблема клиента (конкретная, измеримая)
2. Контекст: почему это было сложно
3. Ваше решение (с техническими деталями)
4. Измеримый результат: +40% конверсия, -60% время загрузки
5. Вывод и применимость для других
```

**Экспертные разборы:**
- "Почему большинство [X] делают [Y] неправильно"
- "Что я узнал после [N] проектов в [нише]"
- Обзор технологии с личной оценкой

**Не делать:**
- Публиковать только на своём сайте
- Писать только продающий контент ("Закажите у нас")
- Публиковать без конкретных цифр и результатов

**Площадки для русскоязычной аудитории:**
- VC.ru — бизнес и технологии
- Хабр — технический контент
- Telegram-канал — кейсы, процесс, разборы
- Medium (EN) — для выхода на международную аудиторию

**Минимальный темп:** 1–2 публикации в месяц на внешних площадках.

### 2.3 🟠 Отзывы клиентов на внешних площадках

Отзыв на вашем сайте = вы говорите о себе.
Отзыв на Clutch / Google = внешний источник говорит о вас.

ИИ различает эти два типа. Второй весит значительно больше.

**Как попросить отзыв (скрипт):**
> "[Имя], рад что проект прошёл успешно! Если у вас есть пара минут, буду благодарен за отзыв на Clutch — это помогает мне находить новых клиентов. Я пришлю прямую ссылку, займёт 5 минут."

**Что просить упомянуть в отзыве:**
- Конкретную задачу которую решали
- Измеримый результат
- Опыт работы с вами (процесс, коммуникация)

**Бэклинки от клиентов:**
Попросить добавить в футер их сайта: `Разработка: [Ваше имя](ссылка на сайт)`

Не нужно публиковать много. Нужно публиковать регулярно и по конкретным темам.

**Месячный цикл (8 публикаций в месяц):**

```
Процесс:
```

## Слой 3 — Мониторинг (как понять что работает)

### 3.1 Инструменты проверки

| Инструмент | Что проверяет | URL |
|---|---|---|
| **Google Rich Results Test** | Правильность JSON-LD schema | search.google.com/test/rich-results |
| **Schema.org Validator** | Синтаксис structured data | validator.schema.org |
| **Google Search Console** | Индексация, ошибки, кликабельность | search.google.com/search-console |
| **Bing Webmaster Tools** | Индексация в Bing / Copilot | bing.com/webmasters |
| **OpenGraph.xyz** | Превью при шаринге | opengraph.xyz |
| **PageSpeed Insights** | Core Web Vitals | pagespeed.web.dev |
| **Ahrefs / Semrush** | Бэклинки, позиции | платные |

### 3.2 Признаки что GEO работает

**Через 1–3 месяца:**
- При вводе вашего имени в ChatGPT / Perplexity — появляется ваш сайт или правильная информация о вас
- Google Search Console показывает рост impressions по информационным запросам

**Через 3–6 месяцев:**
- Ваш сайт начинает цитироваться в ответах Perplexity на нишевые запросы
- Рост брендовых запросов (люди гуглят ваше имя)
- Входящие запросы "нашёл вас через ChatGPT / Perplexity"

**Ручная проверка раз в месяц:**
```
Спросите у ChatGPT, Claude, Perplexity:
- "Что ты знаешь о [ваше имя]?"
- "Найди [профессию] в [нише] с опытом в [специализация]"
- "Топ [профессий] для [целевая аудитория]"
```

```
[ ] og:title, og:description, og:image, og:url, og:locale, og:site_name
```

## Частые ошибки

| Ошибка | Почему плохо | Как правильно |
|---|---|---|
| FAQPage schema без видимой FAQ-секции | Google игнорирует, может понизить | Сначала HTML, потом schema |
| Блокировать GPTBot, OAI-SearchBot, CCBot | Сайт не участвует в AI-ответах | Явно разрешить в robots.txt |
| `sameAs: []` пустой | ИИ не может связать профили | Заполнить после создания профилей |
| Keyword stuffing в description | Поисковики понижают | 1 ключевая фраза, естественный текст |
| Дублировать title и og:title дословно | Упущенная возможность | og:title может быть короче и острее |
| og-image: белый фон + текст | Низкий CTR при шаринге | Фирменный дизайн, крупный текст |
| FAQ только из "что хочу рассказать" | Не соответствует запросам клиентов | Вопросы = то, что спрашивают ИИ |
| Публиковать контент только на своём сайте | Нет внешних сигналов для ИИ | Минимум 1–2 публикации в месяц снаружи |
| Фейковые отзывы | Риск санкций, потеря доверия | Только реальные клиенты |
| Запускать и забыть | GEO требует постоянного обслуживания | Раз в квартал: аудит schema и контента |
